## 1. Validación de viabilidad (Rule of Two)

| Variable | Valor | Fuente |
|---|---|---|
| Ticket medio estimado | ~450€ | Mix fotomatón (350-500€) + algunos combos |
| CVR web (estimado) | 3-5% | Sin datos GA4 aún; benchmark servicios locales |
| Revenue / sesión | €13.5-22.5 | 450€ × 3-5% |
| CPC estimado "fotomaton madrid" | €1.50-2.50 | Nicho local, competencia media |
| **Revenue/sesión vs CPC** | **€13-22 >> €2** | **✅ Google Ads es viable** |

**CAC estimado (escenario moderado):**
- 100 clics × €2 CPC = **€200 gasto**
- CVR 4% = **4 leads**
- Tasa de cierre lead→reserva: **45%** = **1.8 clientes**
- CAC = €200 / 1.8 = **~€110**
- Margen bruto por evento ~55% = **€247 GM**
- CAC/GM = 0.44 → ✅ Rentable desde el primer evento

## 8. Progresión de pujas

```
Semana 1-4 (0-15 conversiones):
  → Maximize Clicks (sin bid cap)
  → Objetivo: datos, no optimizar

Semana 4-8 (15-30 conversiones):
  → Cambiar a Maximize Conversions
  → No tocar presupuesto ni estructura

Mes 2-3 (30+ conversiones):
  → Target CPA: fijar en CPA actual × 1.1
  → Bajar gradualmente hasta CPL objetivo (€40-60)

Campaña de marca:
  → Manual CPC máximo €0.50-1.00 (poco volumen, defensiva)
```

## 11. Checklist de lanzamiento (en orden)

### Pre-lanzamiento (hacer ANTES de gastar 1€)
- [ ] Crear cuenta Google Ads (zona horaria Madrid, moneda EUR)
- [ ] Vincular GA4 con Google Ads
- [ ] Activar auto-tagging en Google Ads
- [ ] **DESACTIVAR** auto-apply recommendations
- [ ] Configurar conversión "generate_lead" (importar desde GA4 o tag directa)
- [ ] Configurar conversión "contact" (WhatsApp click)
- [ ] Activar Enhanced Conversions (gratuito, recupera ~5%)
- [ ] Instalar Google Tag Gateway en Cloudflare (1 clic, mejor tracking)
- [ ] Cambiar conteo conversiones a "Una" (no "Todas")
- [ ] Verificar disparo con Google Tag Assistant
- [ ] Crear las 3 listas de keywords negativas (sección 5)
- [ ] Añadir negativas ANTES de activar la campaña

### Lanzamiento
- [ ] Crear campaña ES_Search_Fotomaton con settings del punto 3
- [ ] Crear los 4 grupos de anuncios con keywords del punto 4
- [ ] Escribir 2-3 RSAs por grupo (punto 6)
- [ ] Añadir todos los assets del punto 7
- [ ] Presupuesto inicial: €20/día
- [ ] Puja: Maximize Clicks
- [ ] Revisar anuncios aprobados (24-48h)

### Primera semana (SOLO monitorizar)
- [ ] Días 1-7: solo añadir negativas diariamente desde Search Terms
- [ ] NO cambiar pujas, presupuesto ni estructura
- [ ] Verificar ≥10 clics/día (si no, revisar keywords o budget)
- [ ] Verificar que las conversiones disparan correctamente

### Optimización semanal (desde semana 2)
- [ ] Revisar Search Terms → añadir negativas (único task semanal primeras 4 semanas)
- [ ] Check CTR tras 100+ clics (objetivo >5% en bodas)
- [ ] Mes 1: NO cambiar pujas (learning period activo)

## 12. Google Tag Gateway (setup Cloudflare — prioritario)

Pimpim usa Cloudflare Pages → setup en 1 clic:

1. Cloudflare Dashboard → tu dominio
2. Zaraz (Web Analytics) → añadir herramienta Google Analytics
3. O bien: Workers → crear worker con el script de Google Tag Gateway

**Por qué importa:** España tiene tasa alta de rechazo de cookies (GDPR + LOPD). Sin Tag Gateway se pierde 15-30% de conversiones. Con Tag Gateway: recuperación completa porque el script va desde tu dominio, no desde googletagmanager.com (que bloquean los ad blockers).

## 13. Conexión con Meta Ads (futuro)

Una vez Google Ads tenga 30+ conversiones/mes y ROAS positivo:

1. Lanzar Meta Ads con **mismas audiencias** que buscan en Google
2. Google Search → intención activa (el usuario ya busca)
3. Meta → crear demanda (usuario no busca pero encaja en perfil boda)
4. Los dos canales se complementan (Meta alimenta branded search en Google)
